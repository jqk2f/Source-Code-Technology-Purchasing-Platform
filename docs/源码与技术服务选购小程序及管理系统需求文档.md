# 源码与技术服务选购小程序及管理系统需求文档

版本：V2.2  
日期：2026-05-26  
定位：源码产品与技术服务展示、客服联系和预约沟通平台  
适用范围：客户移动端、个人管理后台、商品与服务管理、客户预约跟进

## 1. 项目概述

本系统定位为个人主体运营的源码产品、软件成品和技术服务展示平台。客户在移动端浏览平台方上传的源码产品或技术服务，查看价格、技术栈、演示地址、功能介绍、交付内容、购买须知、服务范围和交付标准，并通过客服悬浮窗或预约表单表达沟通意向。

当前版本不做交易履约系统，不涉及在线支付、付款凭证、订单中心、交付管理和售后工单。

## 2. 当前版本边界

### 2.1 当前版本保留

1. 源码产品展示。
2. 技术服务展示。
3. 全局客服悬浮窗，可拖动、可收缩、可弹出联系方式。
4. 预约服务表单。
5. 客户资料编辑，客户可填写称呼、手机号、微信号。
6. 管理后台维护源码产品、技术服务、预约记录和客户资料。

### 2.2 当前版本不做

1. 不做订单模块。
2. 不做支付、收款记录、付款凭证上传。
3. 不做交付管理和售后工单。
4. 不做程序员入驻、认证、竞标、派单、钱包、提现、分账。
5. 不做复杂意向类型，预约记录只保留预约服务这一种业务语义。

## 3. 移动端功能

### 3.1 首页

首页展示推荐源码、热门服务和业务分类。入口聚焦首页、源码、服务、预约、我的。

### 3.2 源码产品

源码产品只保留以下字段：

1. 产品标题。
2. 价格。
3. 技术栈。
4. 状态：草稿、上架、下架。
5. 演示地址。
6. 功能介绍。
7. 交付内容。
8. 购买须知。

客户端只展示状态为上架的源码产品。

### 3.3 技术服务

技术服务只保留以下字段：

1. 服务名称。
2. 服务简介。
3. 起步价。
4. 服务方式。
5. 状态：草稿、上架、下架。
6. 服务范围。
7. 交付标准。
8. 不包含内容。

客户端只展示状态为上架的技术服务。

### 3.4 详情页操作

源码详情页和服务详情页底部只保留两个动作：

1. 客服：打开客服联系方式弹窗。
2. 预约服务：进入预约表单。

预约表单不展示意向类型，不展示来源类型。来源类型和来源 ID 由代码从详情页带入。

### 3.5 预约服务

预约表单字段：

1. 标题。
2. 预约说明。
3. 联系人。
4. 手机号。
5. 微信号。

手机号和微信号至少填写一项。提交后生成预约记录，默认状态为待跟进。

### 3.6 我的

我的页面用于维护客户资料，不展示提交意向、我的意向按钮，不展示公司和客户来源。

客户资料字段：

1. 称呼。
2. 手机号。
3. 微信号。

## 4. 管理后台功能

### 4.1 经营概览

展示今日新增客户、今日预约、待跟进预约、已联系预约。

### 4.2 源码产品管理

维护源码产品的产品标题、价格、技术栈、状态、演示地址、功能介绍、交付内容和购买须知。

状态只保留草稿、上架、下架。

### 4.3 技术服务管理

维护技术服务的服务名称、服务简介、起步价、服务方式、状态、服务范围、交付标准和不包含内容。

状态只保留草稿、上架、下架。

### 4.4 预约管理

预约状态只保留：

1. 待跟进。
2. 已联系。

用户端预约模块和管理端预约模块使用同一套状态颗粒度。

### 4.5 客户管理

客户模块去掉客户来源和公司字段，只维护昵称、手机号、联系人、微信号、状态和备注。

## 5. 验收标准

1. 客服悬浮窗可拖动、可收缩、可弹出联系方式。
2. 我的页面可编辑称呼、手机号、微信号，不出现提交意向、我的意向按钮，不出现客户来源和公司字段。
3. 预约表单不出现意向类型和来源类型。
4. 详情页底部不出现咨询、提交询单、下单登记三套重复动作，只保留客服和预约服务。
5. 用户端和管理端预约状态都只保留待跟进、已联系。
6. 源码和服务后台状态只保留草稿、上架、下架。
7. 客户端只能看到已上架的源码产品和技术服务。
