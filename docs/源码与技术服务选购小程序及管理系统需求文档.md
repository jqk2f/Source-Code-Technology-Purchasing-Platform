# 源码与技术服务选购小程序及管理系统需求文档

版本：V2.0  
日期：2026-05-21  
定位：个人运营的源码产品、软件成品、技术服务展示与选购平台  
适用范围：客户小程序、个人管理后台、商品与服务管理、询单/订单管理、人工收款记录、交付与售后管理

## 1. 项目概述

### 1.1 项目背景

本系统不再定位为多程序员入驻接单平台，而是定位为由个人主体运营的源码、软件成品和技术服务选购平台。平台主要面向有现成源码、系统模板、二开服务、部署服务、技术咨询、问题修复、定制开发等需求的客户。

客户在小程序中浏览平台方上传的产品或服务，查看详情、价格说明、功能截图、演示地址、购买须知和交付方式，并通过提交询单、联系咨询、预约服务或下单登记的方式表达购买意向。平台管理员在后台维护商品、服务、客户、询单、订单、收款记录、交付资料和售后记录。

由于运营主体为个人认证，暂不设计小程序内微信商户支付、资金托管、程序员入驻、认证审核、竞标派单等复杂功能。交易流程以“客户提交意向、平台人工确认、线下或人工收款、后台登记、交付资料、售后跟进”为主。

### 1.2 建设目标

1. 建立源码产品和技术服务的线上展示能力。
2. 支持客户快速筛选、搜索、收藏和咨询产品或服务。
3. 支持客户提交购买意向、需求补充、预约服务或定制询价。
4. 支持管理员在后台上传产品、维护服务、管理价格、配置展示内容。
5. 支持管理员跟进客户询单、确认订单、记录收款、安排交付和处理售后。
6. 在不接入线上商户支付的前提下，实现较完整的交易记录和客户管理。
7. 支持后续升级为企业主体后扩展在线支付、电子合同、自动发货等能力。

### 1.3 产品形态

1. 客户端小程序：用于产品浏览、服务选购、搜索筛选、详情查看、提交咨询、提交购买意向、查看订单/交付/售后。
2. 管理后台 Web：用于商品管理、服务管理、客户管理、询单管理、订单管理、收款记录、交付管理、售后管理、内容运营和数据统计。
3. 服务端接口：提供用户、商品、分类、询单、订单、文件、消息、后台权限等能力。
4. 可选 H5 页面：用于产品分享页、演示说明页、活动页、帮助中心和协议页面。

## 2. 系统边界与不做事项

### 2.1 当前版本不做

1. 不做第三方程序员入驻。
2. 不做程序员实名认证、技能认证、团队认证。
3. 不做需求大厅、多人竞标、平台派单。
4. 不做平台资金托管。
5. 不做小程序内微信商户支付闭环。
6. 不做程序员钱包、提现、分账、保证金。
7. 不做复杂仲裁体系。

### 2.2 当前版本保留

1. 保留产品展示和服务展示。
2. 保留客户咨询、询价、预约和下单登记。
3. 保留后台人工确认订单。
4. 保留后台收款记录，用于记录客户是否已付款、付款方式、付款凭证和确认时间。
5. 保留交付管理，用于记录源码包、网盘链接、部署信息、账号资料、服务结果和交付状态。
6. 保留售后工单和客户跟进。

### 2.3 支付处理原则

当前因个人认证限制，系统不接入小程序在线支付。系统仅做“收款信息展示、付款凭证上传、人工确认收款、后台收款记录”。

可选处理方式：

1. 客户提交购买意向后，系统生成待确认订单，管理员通过电话、微信、企微或其他方式沟通。
2. 管理员向客户提供合规的线下付款方式或收款说明。
3. 客户可上传付款凭证，后台人工核实。
4. 管理员确认收款后，订单进入待交付或服务中。
5. 所有收款方式、票据、税务和平台规则需以实际资质和相关规定为准。

## 3. 用户角色

### 3.1 游客

游客可浏览公开的产品、服务、案例、文章和帮助信息。游客在提交咨询、收藏、查看交付资料、查看订单时需要登录。

### 3.2 客户

客户是购买源码产品或技术服务的用户。客户可浏览商品、筛选分类、查看详情、收藏、咨询、提交购买意向、提交定制需求、上传付款凭证、查看交付记录、申请售后和提交评价。

### 3.3 管理员

管理员即平台个人运营者或其内部协助人员。管理员负责商品上架、服务配置、客户跟进、询单处理、订单确认、收款登记、交付资料维护、售后处理、内容运营和数据查看。

### 3.4 客服/协助人员（可选）

若后续有协助人员，可设置客服角色，负责客户咨询、订单跟进、售后处理，但不允许查看敏感收款信息或修改系统核心配置。

## 4. 业务模式

### 4.1 源码产品售卖

管理员上传源码类产品，包括系统源码、小程序源码、后台管理系统源码、前端模板、后端框架、插件模块、工具脚本等。客户查看产品详情、功能截图、演示地址、技术栈、交付内容和价格后提交购买意向。

### 4.2 成品系统售卖

管理员上传可直接部署或少量修改即可使用的成品系统，例如商城系统、预约系统、会员系统、进销存系统、CRM 系统、工单系统、内容管理系统等。客户可查看演示站、功能清单、部署要求和授权说明。

### 4.3 技术服务选购

管理员发布标准化技术服务，例如源码部署、环境安装、Bug 修复、功能二开、接口对接、小程序上架协助、服务器迁移、数据库处理、代码讲解、技术咨询等。客户选择服务并提交问题说明。

### 4.4 定制需求询价

客户没有合适的现成商品时，可提交定制需求。管理员根据需求进行人工评估，并在后台记录报价、周期、沟通结果和是否转化为订单。

### 4.5 组合套餐

支持将源码产品与技术服务组合销售，例如“源码 + 部署服务”、“源码 + 二开 3 天”、“系统成品 + 域名服务器部署 + 使用培训”。

## 5. 客户端小程序功能需求

### 5.1 首页

1. 展示平台名称、搜索入口、分类导航、推荐产品、热门服务、最新上架、优惠套餐、案例展示。
2. 支持首页 Banner 配置，可跳转产品详情、服务详情、活动页或外部说明页。
3. 支持模块化配置：首页推荐源码、推荐服务、成品系统、技术支持、客户案例、文章教程。
4. 支持未登录浏览公开内容。
5. 支持根据浏览记录展示最近看过和猜你喜欢。

### 5.2 登录与账号

1. 支持微信授权登录。
2. 支持手机号绑定。
3. 支持查看和编辑头像、昵称、手机号、公司名称、联系人、联系地址。
4. 支持账号注销申请。
5. 支持隐私政策和用户协议确认。

### 5.3 分类浏览

1. 支持源码分类，例如小程序源码、Web 系统源码、后台管理系统、移动端源码、插件组件、脚本工具、AI 应用、行业系统。
2. 支持服务分类，例如部署服务、二开服务、Bug 修复、接口对接、数据处理、技术咨询、上架协助。
3. 支持按技术栈筛选，例如 Vue、React、UniApp、微信小程序、Java、PHP、Node.js、Python、Go、MySQL、Redis、Docker。
4. 支持按价格区间、交付方式、是否含部署、是否有演示、上架时间筛选。
5. 支持排序：综合推荐、最新上架、价格升序、价格降序、浏览量、销量/成交量。

### 5.4 搜索

1. 支持搜索产品名称、服务名称、技术栈、行业关键词。
2. 支持搜索历史。
3. 支持热门搜索词。
4. 支持无结果时推荐提交定制需求或联系客服。

### 5.5 产品列表

1. 展示产品封面、标题、简短卖点、技术栈、价格或起售价、是否含部署、是否支持二开、浏览量、成交量。
2. 支持收藏、分享、咨询。
3. 支持产品状态展示：在售、暂停售卖、即将上架、已下架。
4. 支持角标：推荐、热门、新品、可演示、可部署、可二开。

### 5.6 产品详情

1. 展示产品标题、封面、价格说明、技术栈、适用行业、演示地址、演示账号、更新时间。
2. 展示功能清单、页面截图、后台截图、接口说明、数据库说明、部署环境、运行要求。
3. 展示交付内容，例如源码包、数据库脚本、部署文档、使用说明、演示数据、授权说明。
4. 展示购买须知，例如源码售出规则、是否支持退款、是否包含售后、是否包含部署、二开费用说明。
5. 展示常见问题，例如能否商用、能否二开、是否包上线、是否支持远程部署。
6. 支持查看演示、联系客服、提交购买意向、加入收藏、分享。
7. 支持关联服务推荐，例如购买源码时推荐部署服务、二开服务、讲解服务。

### 5.7 服务列表

1. 展示服务封面、服务名称、服务简介、起步价、预计周期、适用场景。
2. 支持筛选服务类型、价格、周期、技术栈。
3. 支持收藏、咨询、预约服务。

### 5.8 服务详情

1. 展示服务内容、服务范围、交付标准、服务流程、所需资料、预计周期、价格说明。
2. 展示不包含内容，避免客户误解。
3. 展示售后说明和响应时间。
4. 支持提交服务预约、上传问题截图、填写补充说明。

### 5.9 购买意向/下单登记

1. 客户点击购买或预约后生成购买意向。
2. 客户填写联系人、手机号、微信号或其他联系方式、购买数量、是否需要部署、是否需要二开、补充说明。
3. 源码产品可选择附加服务，例如部署、环境安装、代码讲解、功能修改。
4. 技术服务需填写问题描述、系统环境、报错截图、访问地址、期望完成时间。
5. 提交后生成待沟通或待确认订单。
6. 系统提示“管理员将联系确认价格、交付方式和付款方式”。

### 5.10 定制需求提交

1. 客户可填写需求标题、行业类型、功能描述、参考产品、预算范围、期望周期、联系方式。
2. 支持上传附件，例如原型图、截图、需求文档、视频说明。
3. 支持选择需求类型：源码二开、系统定制、Bug 修复、部署运维、接口对接、数据处理、技术咨询。
4. 提交后进入后台询单池，由管理员跟进。

### 5.11 订单中心

1. 客户可查看自己的订单或购买意向。
2. 订单状态包括待沟通、待确认、待付款、待确认收款、待交付、服务中、待验收、已完成、售后中、已取消。
3. 订单详情展示商品/服务信息、订单金额、附加服务、联系人、沟通备注、交付状态、售后状态。
4. 对于未确认价格的询单，展示“待管理员报价”。
5. 对于已确认订单，展示付款说明、凭证上传入口和联系客服入口。

### 5.12 付款凭证上传

1. 客户可在订单详情页上传付款截图或转账凭证。
2. 支持填写付款金额、付款时间、付款方式、备注。
3. 上传后订单进入待确认收款状态。
4. 后台确认后客户可看到收款已确认。
5. 系统不自动校验真实到账，以后台人工确认为准。

### 5.13 交付中心

1. 客户可查看已购买产品或服务的交付记录。
2. 交付内容可包括网盘链接、提取码、Git 地址、压缩包说明、部署文档、账号资料、远程服务结果。
3. 敏感交付信息仅订单客户可见。
4. 支持客户确认已收到交付资料。
5. 支持交付资料更新提醒。

### 5.14 售后与反馈

1. 客户可针对订单发起售后。
2. 售后类型包括资料无法下载、源码运行问题、部署问题、功能疑问、服务未完成、其他问题。
3. 支持填写问题描述、上传截图、补充环境信息。
4. 支持查看售后处理进度和处理结果。
5. 支持售后完成后评价服务。

### 5.15 收藏与浏览记录

1. 支持收藏产品和服务。
2. 支持查看浏览历史。
3. 支持取消收藏。
4. 支持根据收藏和浏览推荐相关产品。

### 5.16 消息通知

1. 支持站内消息。
2. 可选微信订阅消息，用于订单状态、收款确认、交付提醒、售后回复。
3. 通知类型包括询单回复、报价确认、付款凭证审核、交付完成、售后处理、系统公告。

### 5.17 帮助中心

1. 展示购买流程。
2. 展示交付说明。
3. 展示源码购买须知。
4. 展示服务边界说明。
5. 展示售后规则。
6. 展示联系方式。

## 6. 管理后台功能需求

### 6.1 后台首页

1. 展示今日新增客户、今日询单、待跟进询单、待确认收款、待交付订单、售后待处理。
2. 展示商品数量、上架商品、下架商品、服务数量。
3. 展示订单金额统计、人工确认收款金额、成交订单数、转化率。
4. 展示最近询单、最近订单、最近售后。
5. 支持按日期查看经营概览。

### 6.2 管理员账号与权限

1. 支持管理员账号登录。
2. 支持角色：超级管理员、运营、客服、交付人员、只读人员。
3. 支持菜单权限和操作权限。
4. 支持操作日志。
5. 支持修改密码和登录日志。

### 6.3 商品分类管理

1. 新增、编辑、删除、启用、禁用分类。
2. 支持多级分类。
3. 支持设置分类图标、排序、首页展示状态。
4. 支持配置分类 SEO/分享标题和描述。

### 6.4 技术标签管理

1. 管理技术栈标签。
2. 管理行业标签。
3. 管理服务标签。
4. 支持标签排序、启用、禁用。
5. 支持热门标签配置。

### 6.5 源码产品管理

1. 新增产品。
2. 编辑产品基础信息：标题、副标题、封面图、轮播图、分类、标签、价格、原价、起售价、库存/可售状态。
3. 编辑产品技术信息：技术栈、前端框架、后端语言、数据库、部署环境、运行要求、源码结构。
4. 编辑产品展示内容：功能介绍、功能清单、页面截图、后台截图、演示地址、演示账号、视频介绍。
5. 编辑交付内容：源码包说明、数据库脚本、部署文档、使用文档、授权说明、是否含演示数据。
6. 编辑购买须知：退款规则、售后范围、是否含部署、是否支持二开、商用说明。
7. 设置产品状态：草稿、上架、下架、售罄、隐藏。
8. 设置推荐属性：新品、热门、首页推荐、分类推荐。
9. 支持复制产品、预览产品、批量上下架。

### 6.6 技术服务管理

1. 新增服务。
2. 编辑服务名称、封面、分类、标签、起步价、服务周期、服务方式。
3. 编辑服务范围、服务流程、客户需提供资料、交付标准、不包含内容。
4. 编辑售后说明、响应时间、加急说明。
5. 设置服务状态：草稿、上架、下架。
6. 支持服务套餐配置，例如基础版、标准版、高级版。

### 6.7 套餐管理

1. 支持将多个产品和服务组合为套餐。
2. 配置套餐名称、套餐价格、包含内容、适用场景、购买须知。
3. 支持设置套餐推荐、限时优惠、上下架。
4. 支持在产品详情页展示关联套餐。

### 6.8 客户管理

1. 查看客户列表。
2. 支持按昵称、手机号、微信、注册时间、成交状态、标签筛选。
3. 查看客户详情，包括基础资料、浏览记录、收藏记录、询单记录、订单记录、售后记录、跟进备注。
4. 支持添加客户标签，例如重点客户、已成交、意向强、需回访、售后中。
5. 支持客户备注和跟进记录。
6. 支持导出客户数据。

### 6.9 询单管理

1. 查看客户提交的购买意向、服务预约和定制需求。
2. 询单状态包括待跟进、已联系、待报价、已报价、已转订单、已失效、已关闭。
3. 支持记录沟通内容、客户预算、需求重点、报价方案、下次跟进时间。
4. 支持将询单转为订单。
5. 支持分配跟进人。
6. 支持设置询单优先级。
7. 支持上传沟通附件。

### 6.10 订单管理

1. 查看订单列表。
2. 订单来源包括产品购买、服务预约、定制需求、后台手工创建。
3. 支持订单状态管理：待沟通、待确认、待付款、待确认收款、待交付、服务中、待验收、已完成、售后中、已取消。
4. 查看订单详情，包括客户信息、商品/服务信息、金额、附加服务、付款说明、凭证、交付资料、售后记录、内部备注。
5. 支持后台修改订单金额、补充费用、折扣金额、最终成交价。
6. 支持后台确认订单、取消订单、完成订单。
7. 支持订单内部备注和客户可见备注分离。
8. 支持订单导出。

### 6.11 收款记录管理

1. 系统不做自动支付，仅记录人工收款。
2. 支持客户上传付款凭证。
3. 管理员查看凭证并确认是否到账。
4. 支持记录收款金额、收款方式、收款时间、收款账号、交易流水号、备注。
5. 收款方式可配置为微信转账、支付宝转账、银行卡转账、对公转账、其他。
6. 支持一笔订单多次收款。
7. 支持收款作废、修正和备注。
8. 支持收款统计和导出。

### 6.12 交付管理

1. 支持为订单添加交付记录。
2. 交付类型包括源码文件、网盘链接、Git 仓库、部署信息、账号密码、文档资料、远程服务记录。
3. 支持设置交付内容客户可见或仅后台可见。
4. 支持交付状态：待交付、部分交付、已交付、需补充。
5. 支持交付资料更新和客户通知。
6. 支持客户确认收到资料。
7. 支持敏感信息加密或脱敏展示。

### 6.13 文件资源管理

1. 管理商品图片、截图、文档、交付附件。
2. 支持上传、删除、分类、搜索。
3. 支持文件大小和格式限制。
4. 支持私有文件和公开文件区分。
5. 私有交付文件需要登录且订单匹配后才能访问。

### 6.14 售后管理

1. 查看售后工单列表。
2. 售后状态包括待处理、处理中、待客户反馈、已解决、已关闭。
3. 支持记录问题描述、截图、处理方案、处理结果。
4. 支持关联订单和交付记录。
5. 支持售后超时提醒。
6. 支持常见问题沉淀到知识库。

### 6.15 评价管理

1. 客户可对产品或服务进行评价。
2. 后台可查看评价、回复评价、隐藏违规评价。
3. 评价维度包括产品质量、交付完整度、服务态度、响应速度、性价比。
4. 支持评价展示开关。

### 6.16 内容管理

1. Banner 管理。
2. 首页模块管理。
3. 公告管理。
4. 帮助中心文章管理。
5. 购买须知管理。
6. 售后规则管理。
7. 技术文章/教程管理。
8. 分享海报和分享文案管理。

### 6.17 营销配置

1. 支持优惠展示，例如限时优惠、套餐优惠、老客户优惠。
2. 支持优惠券展示和后台登记，但不与在线支付自动核销。
3. 支持客户来源标记，例如自然访问、朋友推荐、短视频、社群、公众号。
4. 支持活动页配置。

### 6.18 消息管理

1. 管理站内信模板。
2. 管理订阅消息模板。
3. 支持发送订单状态通知、收款确认通知、交付通知、售后通知。
4. 支持消息发送记录和失败记录。

### 6.19 数据统计

1. 商品浏览量、收藏量、咨询量、询单量、成交量。
2. 服务浏览量、预约量、成交量。
3. 客户新增、活跃、成交、复购。
4. 询单转化率、订单成交率、售后率。
5. 人工确认收款金额、订单金额、未收款金额。
6. 热门分类、热门搜索词、热门商品。
7. 支持按日期、分类、商品、来源筛选。

### 6.20 系统配置

7. 配置隐私政策、用户协议、源码购买协议。

## 7. 核心业务流程

### 7.1 源码产品购买流程

客户浏览产品列表，进入产品详情，查看功能、截图、演示地址、价格和购买须知。客户提交购买意向并填写联系方式、是否需要部署、是否需要二开。后台管理员收到询单后联系客户确认细节和最终价格。客户按约定方式付款并上传付款凭证。管理员确认收款后，在后台添加交付资料。客户查看交付内容并确认收到。订单完成后进入售后期。

### 7.2 技术服务预约流程

客户选择技术服务，填写问题说明、系统环境、截图和期望时间。后台管理员评估服务范围和费用，转为订单。客户付款后管理员开始服务。服务完成后管理员填写服务结果和交付说明，客户确认完成并评价。

### 7.3 定制需求询价流程

客户提交定制需求，后台生成询单。管理员沟通需求、评估成本、记录报价和周期。客户确认后转为订单。若需求复杂，可拆分为阶段交付，但当前版本不做资金托管，仅在后台记录阶段收款和阶段交付。

### 7.4 收款确认流程

客户上传付款凭证，订单进入待确认收款。管理员核实到账后记录实际收款金额、方式和时间。若金额不足，可标记部分收款；若凭证无效，可驳回并填写原因。确认收款后订单进入待交付或服务中。

### 7.5 交付流程

管理员根据订单添加交付记录，包括源码文件、网盘链接、部署说明、账号信息或服务结果。客户在订单详情查看交付内容。若交付内容需要补充，管理员可追加交付记录并通知客户。

### 7.6 售后流程

客户在售后期内提交问题，管理员查看订单和交付资料，判断是否属于售后范围。属于售后范围则处理并记录结果；超出售后范围则可转为新的技术服务询单。

## 8. 订单状态设计

1. 待沟通：客户提交意向后，等待管理员联系。
2. 待确认：管理员已沟通，等待客户确认购买内容或报价。
3. 待付款：订单已确认，等待客户付款。
4. 待确认收款：客户已上传付款凭证，等待管理员核实。
5. 部分收款：订单只收到部分款项。
6. 待交付：已确认收款，等待管理员交付资料。
7. 服务中：技术服务正在处理。
8. 待验收：服务或交付已完成，等待客户确认。
9. 已完成：客户确认或管理员完成订单。
10. 售后中：订单存在售后工单。
11. 已取消：客户或管理员取消订单。
12. 已关闭：订单失效、超时或无需继续跟进。

## 9. 数据对象需求

### 9.1 客户

### 9.2 商品

### 9.3 服务

字段包括服务 ID、名称、分类、标签、起步价、服务周期、服务范围、服务流程、所需资料、交付标准、售后说明、状态、排序。

### 9.4 询单

字段包括询单 ID、客户 ID、来源类型、关联商品/服务、需求描述、联系方式、预算、期望时间、附件、状态、跟进人、跟进记录、报价金额、下次跟进时间。

## 10. 权限与安全需求

1. 客户只能查看自己的订单、凭证、交付资料和售后记录。
2. 公开商品图片和介绍可公开访问，源码文件、网盘链接、账号信息等交付资料必须受权限保护。
3. 后台不同角色按权限查看菜单和数据。
4. 收款记录、交付资料、客户联系方式属于敏感信息，需限制访问并记录操作日志。
5. 文件上传需校验大小、格式和安全风险。
6. 管理员删除商品、删除交付资料、作废收款记录等操作需二次确认。
7. 登录、订单、收款、交付、售后等关键操作需记录日志。
8. 客户手机号、微信号、付款凭证等敏感信息建议脱敏展示。

## 11. 非功能需求

### 11.1 性能

1. 小程序首页首屏加载建议小于 2 秒。
2. 商品列表接口响应建议小于 500 毫秒。
3. 图片需压缩和懒加载。
4. 商品详情中的截图较多时应分页或按需加载。

### 11.2 可用性

1. 商品展示、询单提交、订单查看、交付查看为核心功能，需保证稳定。
2. 客户提交询单后必须有明确反馈。
3. 后台待办事项需突出显示，避免漏跟进。

### 11.3 扩展性

1. 预留在线支付字段和支付状态，便于后续企业认证后接入。
2. 预留电子合同、自动发货、会员、优惠券自动核销能力。
3. 分类、标签、状态、付款方式、交付类型应后台可配置。

### 11.4 合规与提示

1. 源码售卖需明确授权范围、商用范围、退款规则和售后边界。
2. 不应售卖侵权、盗版、违法违规或灰产相关源码和服务。
3. 客户付款和开票等事项需以实际主体资质为准。
4. 隐私政策需说明客户联系方式、订单信息、付款凭证和交付信息的使用目的。

## 15. 验收标准

1. 管理员可在后台新增并上架源码产品。
2. 管理员可在后台新增并上架技术服务。
3. 客户可在小程序浏览、搜索、筛选产品和服务。
4. 客户可提交购买意向、服务预约和定制需求。
5. 管理员可跟进询单并转为订单。
6. 管理员可记录人工收款和确认付款凭证。
7. 管理员可为订单添加交付资料。
8. 客户可查看自己的订单、收款状态、交付资料和售后进度。
9. 系统不依赖在线支付也能完整记录业务过程。

## 16. 风险与注意事项

1. 个人认证主体不适合强依赖在线支付、分账、托管等能力，当前版本应以询单和人工确认为主。
2. 源码产品售出后复制成本低，需在购买须知中明确售后和退款边界。
3. 源码授权范围必须写清楚，例如是否允许商用、是否允许二次销售、是否允许转卖。
4. 演示账号和交付资料可能包含敏感信息，需做好权限控制。
5. 技术服务范围容易扩大，服务详情和订单备注中必须写明包含与不包含内容。
6. 后续如果接入支付，应重新梳理主体资质、支付商户号、退款、发票、税务和平台规则。
